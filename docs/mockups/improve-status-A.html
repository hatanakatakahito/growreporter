案 A: 「着手する」ボタン前面化（最小介入）

既存のテーブル UI を保ちつつ、ステータス列を「操作」として前面化。 draft アイテムに primary 色の「着手する」ボタンを配置、クリックで in_progress 遷移 + Before スナップショット取得が自動で走る。 対応中アイテムには「完了報告」ボタンを表示。完了後は静的なバッジ表示に戻す。

実装範囲: Improve.jsx のテーブル行レンダリング部のみ（ステータス列を条件分岐でボタン/バッジに切替）。大きな構造変更なし。

現行との違いのポイント
  • draft 時: グレーの `<select>` ドロップダウン → primary 色の「着手する」大ボタン
  • ボタン直下に「着手と同時に現ページ状態を自動記録します」説明文 → 対応中経由の意味を伝える
  • in_progress 時: 行全体が淡い amber 背景 → 「作業中」が視覚的にわかる
  • in_progress 時: 緑の「完了にする」ボタン → 次のアクションが明確
  • completed 時: 計測中・計測完了をカード状態表示 → 「選択肢」ではなく「結果」として見える
  • draft 時の ドロップダウン選択肢から「完了」を除外 → 起案→完了の直行を UI 上で封じる（誘導ダイアログも残す）
