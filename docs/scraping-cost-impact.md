# GA4上位100ページスクレイピング機能のコスト影響

## 概要

このドキュメントは、GA4上位100ページスクレイピング機能の導入による **Firebase / Google Cloud のコスト変化** を整理したものです。価格は目安であり、リージョン・時期により変動します。

---

## 1. コストが増える要因

### 1.1 Cloud Functions（実行時間・メモリ）

| 項目 | 内容 | 目安コスト |
|------|------|------------|
| **scrapeTop100Pages**（手動・トリガー・スケジュール） | 2GiBメモリ、最大約9分（540秒） | 1回あたり約 **$0.01〜0.02**（実行時間により変動） |
| **siteCreatedSheetsExport**（サイト登録時） | 従来 5分→**10分**、512MiB→**2GiB** に変更 | 1サイト登録あたり **+$0.01 前後**（スクレイピング分） |
| **updatePageScrapingDataScheduled**（毎日3時） | 2GiB、最大10分。対象サイトがいる日のみ長時間実行 | 対象日 1回あたり **約 $0.01〜0.02** |

**Functions の課金の考え方（Gen2）**

- **GB秒**: メモリ(GB) × 実行時間(秒) で課金
- 例: 2GB × 300秒 = 600 GB秒 → 約 $0.0015（単価により変動）
- 無料枠: 月 200万 GB秒程度（小規模ならほぼ収まる想定）

---

### 1.2 Firestore（読み書き・削除）

**1サイトあたり 1回のスクレイピングで発生する主な操作**

| 操作 | 回数（目安） | 備考 |
|------|--------------|------|
| 読み取り | 約 100〜150 | サイト取得、トークン、既存 pageScrapingData 取得、進捗参照など |
| 書き込み | 約 110〜120 | 進捗 2回、pageScrapingData 約100件、meta 1回、エラーログ数件 |
| 削除 | 0〜100 | 再実行時のみ（既存 pageScrapingData の削除） |

**月間の目安（例）**

- 新規サイト登録: 5サイト/月 → 約 500 読 + 550 書 + 0 削除（初回のため）
- 手動実行: 10回/月 → 約 1,200 読 + 1,150 書 + 1,000 削除
- 定期実行: 30サイトを30日でローテーション → 約 3,000 読 + 3,300 書 + 3,000 削除

**Firestore 料金目安**

- 読み取り: $0.06 / 10万回
- 書き込み: $0.18 / 10万回
- 削除: $0.02 / 10万回  

上記の月間例だと、**月 $0.01〜0.05 程度** の増加にとどまる想定です。

---

### 1.3 ネットワーク（送信）

- スクレイピングは **外部サイトへの HTTP リクエスト（送信）** が 100回/サイト程度発生します。
- Cloud Functions からの送信は従量課金ですが、HTML 取得程度のトラフィックでは **月数ドル以下** の規模です。

---

## 2. 増えない・ほぼ変わらないもの

- **Cloud Storage**: スクレイピング結果は Firestore のみで、Storage は使用していません。
- **Authentication**: 既存と同様で、スクレイピングによる増加はありません。
- **Hosting**: フロントのみのため変化なし。

---

## 3. 月間コストの目安（増分のみ）

| 利用パターン | 想定 | 月あたりの増分（目安） |
|--------------|------|-------------------------|
| 小規模 | サイト登録 2回/月、手動 2回/月、定期 2サイト/月 | **約 $0.05〜0.15** |
| 中規模 | サイト登録 5回/月、手動 10回/月、定期 10サイト/月 | **約 $0.20〜0.50** |
| 大規模 | サイト登録 15回/月、手動 30回/月、定期 30サイト/月 | **約 $0.50〜1.50** |

（Functions の実行時間が短い場合は下振れ、長い場合は上振れします。）

---

## 4. 無料枠との関係

- **Cloud Functions**: 月 200万 GB秒などの無料枠があり、スクレイピングだけなら **小〜中規模ではほぼ無料枠内** に収まる可能性が高いです。
- **Firestore**: 日 5万回読み取り・2万回書き込みなどの無料枠があり、**同様に小〜中規模では無料枠内** が期待できます。
- いずれも **利用サイト数・実行回数** が増えると、無料枠を超えて課金が発生します。

---

## 5. コストを抑えるための設計（実装済み）

1. **定期実行は 1日1サイトまで**  
   `MAX_SITES_PER_RUN = 1` により、長時間・高頻度の実行を抑制。
2. **手動実行のレート制限**  
   同一サイトは 1時間に1回まで（`scrapeTop100Pages` 側で制限）。
3. **データは Firestore のみ**  
   Cloud Storage を使わず、ストレージコストを抑える設計。
4. **最新データのみ保持**  
   再スクレイピング時に古い `pageScrapingData` を削除し、ドキュメント数を抑えている。

---

## 6. まとめ

- **増加する主なコスト**: Cloud Functions の実行時間・メモリ、Firestore の読み書き・削除。
- **目安**: 小規模利用で **月あたり約 $0.05〜0.15**、中規模で **約 $0.20〜0.50** 程度の増加を見込む想定です。
- 無料枠内に収まるかは利用量次第ですが、**サイト数・実行回数が少ないうちは無料枠でほぼカバーできる** 設計になっています。
- 正確なコストは [Firebase 料金](https://firebase.google.com/pricing) と [Cloud Functions 料金](https://cloud.google.com/functions/pricing) でご確認ください。
