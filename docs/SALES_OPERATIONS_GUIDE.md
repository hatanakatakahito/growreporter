# 営業・バックオフィス運用ガイド — 顧客アカウント発行フロー

本ドキュメントは、grow-reporter の顧客アカウント発行・契約管理を **営業・バックオフィス・管理者** が運用するための手順書です。

最終更新: 2026-04-30（v5.9.0 リリース時）
対象: GrowGroup 株式会社 社内メンバー

---

## はじめに

### grow-reporter のプラン体系

| プラン | 月額（税別） | サイト上限 | 主な機能 |
|---|---|---|---|
| 無料 | ¥0 | 1 サイト | データ閲覧 / 基本通知 |
| ビジネス | ¥49,800 | 3 サイト | AI 機能フル / 改善提案 / レポート / 効果測定 |
| 追加サイトオプション | +¥15,000/サイト | — | ビジネスプランに追加可能（1サイトずつ） |

### grow-reporter と board の役割分担

- **grow-reporter**: 顧客がログインして使う Web アプリ。サイト分析・AI 機能を提供
- **board**: 見積・請求書発行・契約管理（請求書 SaaS）

両者の関係:
- 顧客アカウント = grow-reporter で管理
- 見積書・契約書・請求書 = board で発行
- アクティブな契約状態を双方に反映する必要あり

## 4 つのシナリオ早見表

| シナリオ | 何をする？ | どこから始める？ | 詳細 |
|---|---|---|---|
| **A. 顧客が自分で signup** | 何もしない（自動完結） | 顧客が grow-reporter のフォームに入力 | §A |
| **B. 代行で無料アカウント発行** | admin がサイレント作成 + 後送信 | `/admin/users` | §B |
| **C. 代行で有料アカウント発行（board 起点）** | board 取り込み + サイレント作成 + 後送信 | board → `/admin/inquiries` | §C |
| **D. 既存無料 → 有料アップグレード（board 起点）** | board 取り込み（既存ユーザー自動マッチ） | board → `/admin/inquiries` | §D |

## §A. シナリオ A: 顧客が自分で signup

### 手順
1. 顧客が grow-reporter の signup フォーム（無料）or 「ビジネスプラン申込」フォームから入力
2. ユーザーアカウントが自動作成
3. 即ウェルカムメールが顧客に送信される
4. ビジネスプラン申込の場合、自動で board に見積書が作成される（管理者宛通知）

### 営業・バックオフィスのアクション
- 営業: ビジネスプラン申込なら admin に「contract_sent」ステータス変更を依頼
- admin: 契約完了したら `/admin/inquiries` で該当 inquiry のステータスを「契約中」に変更 → users.plan='business' が自動反映

---

## §B. シナリオ B: 代行で無料アカウント発行

例: 「Web 制作納品時に grow-reporter の無料アカウントを発行して引き渡す」

### 手順

#### 1. 管理画面でユーザー作成（サイレント）
1. `/admin/users` を開く
2. 「新規ユーザー作成」ボタン
3. フォーム入力:
   - メールアドレス（必須）: 顧客のメール
   - 氏名: 顧客の代表者名
   - 組織名: 顧客の会社名
   - 電話番号: ハイフンなし
   - プラン: **free**
   - パスワード: 任意（空欄でも可）
   - **「ウェルカムメールを送信する」のチェックを外す** ← 重要！

4. 「作成」ボタン → ユーザー作成完了（メールは送信されない）

#### 2. 代行サイト登録（必要なら）
1. 作成されたユーザーの `/admin/users/{uid}` を開く
2. 「サイトを追加」ボタン
3. 顧客のサイト URL・GA4・GSC を設定

#### 3. アカウント情報を顧客に送信
1. `/admin/users/{uid}` を開く
2. **「アカウント情報をメールで送信」ボタン** をクリック
3. 確認ダイアログで「OK」
4. 顧客にパスワードリセットリンク + ログイン情報のメールが送信される
5. ボタン下に「最終送信: YYYY/MM/DD HH:mm」が表示される

#### 4. 顧客側のアクション
- メール受信
- パスワード設定リンクをクリック
- パスワード設定 → ログイン
- すでにサイトが登録された状態でダッシュボードが開く

### よくあるミス
- ❌ ウェルカムメール送信のチェックを外し忘れて、準備未完了の状態でメールが送信される
- ❌ 「アカウント情報をメールで送信」ボタンを押し忘れる（顧客がいつまでも知らない）
- ❌ 代行サイト登録を忘れて、顧客がログイン後に「何も設定されていない」と困惑

## §C. シナリオ C: 代行で有料アカウント発行（board 起点）

例: 「営業がビジネスプランの提案 → board で先に見積を作成 → 顧客承認後にアカウント発行」

### 前提
- 営業が **board で見積書を作成済み** であること
- 顧客承認が取れていること（成約）

### 手順

#### 1. board 取り込み
1. `/admin/inquiries` を開く
2. **「board から取り込み」ボタン** をクリック
3. board の **案件 ID** を入力（board の案件詳細 URL の末尾の数字）
4. 「プレビューを取得」 → 内容確認
   - 顧客情報（組織名・担当者・メール・住所）
   - 契約期間
   - 自動判別された申込種別（メイン契約 + 追加サイト など）
   - 見積明細
   - 既存ユーザーマッチ状況

#### 2. ユーザー作成オプションの選択
- 「ユーザーアカウントも同時に作成する」 **チェック ON**（admin ロール限定）
- ※ アカウント作成後、メールは送信されません（後で別途送信）

#### 3. 取り込み実行
1. 「取り込みを実行」ボタン
2. 完了画面でユーザー詳細へのリンクが表示される

#### 4. 代行サイト登録（必要なら）
- §B の手順 2 と同様

#### 5. アカウント情報送信
- §B の手順 3 と同様

#### 6. プラン反映（contract active 化）
1. `/admin/inquiries` で該当 inquiry を開く
2. ステータスドロップダウンを **「契約中」** に変更
3. 確定 → users.plan='business' が自動反映 + extraSitesCount も反映 + board 見積書が自動再構築（再構築型ロジック）

### よくあるミス
- ❌ 「ユーザーも作成」のチェックを忘れて、後から手動でユーザー作成 → 紐付けの二度手間
- ❌ プラン反映前にメール送信して顧客がログインしても無料プランのまま見える
- ❌ 同じ board 案件を 2 度取り込もうとして「重複エラー」になる

## §D. シナリオ D: 既存無料 → 有料アップグレード（board 起点）

例: 「既に無料で使っている顧客に営業 → ビジネスプラン契約成立」

### 前提
- 顧客が grow-reporter で無料アカウントを既に持っている
- 営業が board で見積書を作成済み

### 手順

#### 1. board 取り込み
- §C の手順 1 と同様
- プレビュー画面で **「既存 grow-reporter ユーザーが見つかりました」** と表示されることを確認

#### 2. mergeStrategy の選択（既存 extras がある場合のみ）
顧客が既に追加サイトオプションを持っている場合のみ表示:
- **加算**: 既存 + 取り込み = 新合計
- **上書き**: 取り込みの値で置き換え
- **最大値**: 既存と取り込みの大きい方

通常は「加算」が推奨。

#### 3. 取り込み実行
- 「ユーザーも作成」のチェックは **OFF**（既存ユーザーがあるため）
- 「取り込みを実行」 → inquiry のみ作成

#### 4. プラン反映
1. `/admin/inquiries` で該当 inquiry を「契約中」に変更
2. users.plan: free → business に自動変更
3. プラン変更通知メールが顧客に自動送信される（既存ロジック）

### 注意
- 顧客は既にログイン情報を持っているので、§16 の「アカウント情報をメールで送信」は **不要**

## トラブルシューティング

### Q1: メールが届かない
- スパムフォルダを確認
- 送信元 `alert@grow-reporter.com` がブロックされていないか
- 顧客のメールアドレスが正しいか admin 画面で確認
- 必要なら「アカウント情報をメールで送信」を再度押して再送信（何度でも可）

### Q2: パスワードリセットリンクが期限切れ（72 時間経過）
- `/admin/users/{uid}` で「アカウント情報をメールで送信」を再度押す → 新しいリンクが発行される

### Q3: board と grow-reporter のステータスが食い違う
- board で契約終了したのに grow-reporter で plan='business' のまま:
  - admin が `/admin/inquiries` で該当 inquiry のステータスを「解約」に変更
- grow-reporter で「解約」したが board の見積書がそのまま:
  - 月単位精算ルールで翌月以降の課金は止まる（自動）
  - メインプラン解約の場合のみ、admin が board で手動で「契約終了」処理

### Q4: ユーザー作成と inquiry 紐付けに失敗した
- inquiry 詳細ダイアログで「ユーザー未紐付け」警告バナーが表示される
- email 検索で既存ユーザーを見つけて紐付け
- 該当ユーザーがいなければ `/admin/users` で新規作成 → 戻って email で検索 → 紐付け

### Q5: 同じ顧客から 2 回目の追加サイト申込が来た
- 通常通り `/admin/inquiries` で該当 inquiry をステータス変更
- board の見積書は自動的に「現在の総 extras」で再構築される（明細が累積しない）

### Q6: 解約後の board 見積書はどうなる？
- addon_only 解約: board 見積書も自動再構築（追加サイト明細が消える or 数量更新）
- new_business 解約（プラン解約）: board 見積書は **admin が手動で「契約終了」処理** が必要

## FAQ

### Q: 無料ユーザーに後から営業を仕掛けたい
→ シナリオ D を参照（board で見積作成 → 取り込み → 自動マッチ）

### Q: 複数サイトの顧客を扱いたい
→ board 見積に「追加サイトオプション × N」明細を含める → 取り込みで自動判別

### Q: メールが届く前にパスワードを伝えたい
→ adminCreateUser で `password` 指定 + ウェルカムメール送信チェックを ON にする（§B の Pattern: パスワード即発行）

### Q: 顧客がメールアドレスを変更した
→ admin が users.email を更新 → 「アカウント情報をメールで送信」を再送

### Q: 1 つの顧客で複数の board 案件を扱いたい
→ それぞれの案件 ID で取り込む（重複チェックは boardProjectId 単位）

### Q: 取り込んだ inquiry を間違えて作った
→ `/admin/inquiries` で「申込キャンセル」ステータスに変更（CSV エクスポート前なら削除も可能）

## board 起点フロー × フォーム起点フロー の使い分けガイド

| 観点 | フォーム起点 | board 起点 |
|---|---|---|
| 顧客が grow-reporter を知っている | ✅ 推奨 | △ |
| 営業が顧客と契約交渉済 | △ | ✅ 推奨 |
| 既に board に顧客情報がある | △ | ✅ 推奨 |
| 顧客との連絡を電話・メール中心 | △ | ✅ 推奨 |
| 顧客自身に grow-reporter を体験してもらう | ✅ 推奨 | △ |

基本方針: **板で見積を作成済なら board 起点、フォーム送信済ならフォーム起点** で運用。

## サイト引き渡し / 取り戻しフロー (v5.17.0+)

当社で代行作成したサイトを顧客 (新規 or 既存) に「自分のサイト」として引き渡す機能。

### 基本フロー

#### 新規顧客への引き渡し
1. `/admin/users/{顧客 uid}` を開く (または `/admin/users/` から「新規ユーザー作成」)
2. 「サイト引き渡し」セクションの **「サイトを引き渡す」** ボタンをクリック
3. SiteTransferModal が開く:
   - **既存検索タブ**: メールアドレスで顧客検索 → 候補から選択
   - **新規作成タブ**: 顧客情報入力 → サイレントでアカウント作成
4. 移管対象サイトをチェックボックスで選択 (~10 サイト程度想定)
5. プレビュー画面で警告 (プラン超過 / OAuth 代行運用) を確認 → 「引き渡しを実行」
6. 完了 → 新規作成顧客の場合は **「アカウント情報をメール送信」** ボタンで通知

#### 既存顧客への引き渡し
- 上記と同じ流れだが、ステップ 4 の完了時に顧客に「サイトが追加されました」通知メールが自動送信される

### OAuth 連携の扱い

**当社の Google アカウントが引き続き保持** (`ga4TokenOwner` / `gscTokenOwner` は admin uid のまま):
- 顧客側で再連携不要、データ取得は当社代行運用で継続
- 顧客側 SiteDetail に「OAuth 連携: 当社が代行運用中」バッジ表示
- 顧客が自分のアカウントに切替たい場合: アカウント設定で OAuth 連携 → サイト詳細の「自分の Google で再連携」ボタン

### 取り戻し (誤操作 / サポート対応)

1. `/admin/users/{顧客 uid}` を開く
2. 「あなたが過去に引き渡したサイト: N 件」表示の横にある **「サイトを取り戻す」** ボタン
3. 確認モーダル → 取り戻し実行
4. サイトが admin (自分) に戻り、顧客のダッシュボードから消える

### 顧客退会時の自動処理

`deleteUser` callable で顧客を削除すると:
- `_transferredFromUid` (移管されたサイト) → 自動的に元 admin に逆移管 (削除されない)
- `_transferredFromUid` 無し (顧客が自分で作成したサイト) → 既存通り削除

### 一覧確認

- `/admin/sites` (サイト管理) — 行に「移管済」バッジが表示される
- `/admin/logs` (アクティビティログ) — 「サイト所有権移管 (引き渡し)」「サイト所有権取り戻し」フィルタで履歴閲覧
